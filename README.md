# Seotda Console Game

Windows Console API와 C/C++로 구현한 **섯다 카드 게임**입니다. 20장의 화투패를 섞어 사용자와 4명의 컴퓨터 플레이어에게 두 장씩 배분하고, 두 차례의 베팅과 족보 비교를 거쳐 승자를 결정합니다.

GUI 라이브러리 없이 콘솔의 좌표 이동과 색상 제어만으로 카드 테이블, 패, 판돈, 소지금, 선택 메뉴를 한 화면에 구성했습니다. 단순히 결과만 출력하는 프로그램이 아니라 **입력 → 턴 진행 → 베팅 → 패 공개 → 승패 판정 → 정산**으로 이어지는 한 판의 흐름을 직접 설계하고 구현한 프로젝트입니다.

## 주요 기능

- 1명의 사용자와 4명의 컴퓨터 플레이어가 참여하는 5인 게임
- 1월부터 10월까지 두 장씩 구성된 20장 패 생성 및 무작위 셔플
- 첫 번째 패와 두 번째 패를 순차적으로 공개하는 라운드 진행
- 방향키와 Enter를 이용한 베팅 메뉴 조작
- 다이, 따당, 콜, 하프 베팅 처리
- 플레이어별 콜·다이 상태 및 턴 순서 관리
- 판돈, 콜 비용, 플레이어별 소지금 계산과 승자 정산
- 광땡, 땡, 알리·독사 등의 특수 족보와 끗 족보 판정
- 전원 다이에 따른 기권승 및 최종 패 비교에 따른 승리 처리
- 콘솔 좌표, 문자, 색상을 활용한 카드 테이블 UI

## 게임 진행 흐름

```text
20장 패 생성 및 셔플
        ↓
플레이어별 2장 배분 및 족보 계산
        ↓
첫 번째 패 공개 → 1차 베팅
        ↓
두 번째 패 공개 → 2차 베팅
        ↓
남은 플레이어의 족보 비교
        ↓
승자 결정 및 판돈 정산
```

각 플레이어의 `call`, `die`, 선행 베팅 여부를 구조체로 관리하고, 전체 콜 인원과 다이 인원을 함께 추적해 다음 라운드 진입 또는 기권승 여부를 판단합니다. 컴퓨터 플레이어는 현재 베팅 단계에서 허용되는 행동 중 하나를 무작위로 선택합니다.

## 구현 포인트

### Windows Console API 기반 UI

`SetConsoleCursorPosition`으로 출력 위치를 제어하고 `SetConsoleTextAttribute`로 정보별 색상을 구분했습니다. 화면을 줄 단위로 계속 출력하는 대신 필요한 좌표를 갱신해 콘솔에서도 고정된 게임 보드처럼 보이도록 구성했습니다.

### 카드와 족보의 수치화

20장의 패를 정수로 표현하고 두 패의 조합을 우선순위 값으로 변환했습니다. 승부 시에는 살아 있는 플레이어의 우선순위 값을 비교하므로 화면 출력 로직과 승패 판정 로직을 분리할 수 있었습니다.

### 상태 기반 턴 및 베팅 처리

턴 번호를 플레이어 수로 나눈 값으로 현재 차례를 결정합니다. 베팅 결과에 따라 판돈과 콜 비용을 갱신하고, 콜 또는 다이 상태를 누적해 베팅 종료 조건을 판정하도록 구현했습니다.

### 동적 게임 정보 표시

플레이어의 패, 족보, 판돈, 콜 비용, 소지금을 게임 진행에 맞춰 같은 화면에서 갱신합니다. 금액은 천 단위 구분 기호와 색상을 적용해 가독성을 높였습니다.

## 조작 방법

| 입력 | 동작 |
| --- | --- |
| `←` / `→` | 베팅 항목 이동 |
| `Enter` | 선택한 항목 실행 |

## 기술 스택

- C/C++
- Windows Console API
- Visual Studio 2022
- Windows SDK 10.0

## 실행 방법

1. Windows에서 `Project1.sln`을 Visual Studio 2022로 엽니다.
2. 실행 구성을 `Debug` 또는 `Release`, 플랫폼을 `Win32` 또는 `x64`로 선택합니다.
3. 프로젝트를 빌드한 뒤 실행합니다.

콘솔의 한글과 테두리 문자가 올바르게 표시되도록 실행 환경의 글꼴 및 코드 페이지 설정이 필요할 수 있습니다.

## 프로젝트를 통해 배운 점

- 카드 게임 규칙을 조건문과 우선순위 값으로 모델링하는 방법
- 여러 플레이어의 턴과 행동 상태를 일관되게 관리하는 방법
- 포인터, 동적 메모리, 구조체 배열을 실제 게임 데이터에 적용하는 방법
- 입력 처리, 게임 규칙, 화면 출력을 함수 단위로 나누어 구성하는 방법
- Windows Console API를 이용해 좌표 기반 인터페이스를 만드는 방법

## 현재 범위 및 개선 계획

현재 저장소의 실행 흐름은 5인 게임에 맞춰져 있으며, 타이틀 화면의 인원 선택 기능은 코드에 작성되어 있지만 비활성화된 상태입니다. 또한 올인 메뉴는 표시되지만 실제 금액 처리 로직은 아직 구현되지 않았습니다.

향후에는 다음 항목을 개선할 수 있습니다.

- 올인 및 잔액 부족 상황 처리
- 타이틀 화면과 2~5인 인원 선택 기능 활성화
- 족보 판정 조건의 테이블화 및 단위 테스트 추가
- 컴퓨터 플레이어가 패와 판돈을 고려하도록 의사결정 로직 개선
- 전역 상태 축소와 게임·플레이어·카드 책임의 클래스 분리
- 무승부 및 특수 족보 간 예외 규칙 보완
